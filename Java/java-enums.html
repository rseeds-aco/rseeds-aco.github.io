① What even is an Enum?
🎯
The "Closed Set" Problem
motivation
The problem with plain ints and Strings: Nothing stops someone from passing 42 or "SUNDAE" where you expected a day of the week. The compiler can't save you.
Enums define a fixed, named set of values — like a custom type whose only legal values you list yourself. The compiler enforces it everywhere.
Mental model: "I want a variable that can only ever be NORTH, SOUTH, EAST, or WEST — and nothing else in the universe."
// ❌ Old fragile approach
int direction = 3; // What does 3 mean??
// ✅ Enum approach
Direction dir = Direction.NORTH; // Crystal clear ✨
② Declaring your first Enum
📝
Basic Syntax
simple
Replace class with enum — Java does the rest. Constants live in ALL_CAPS by convention (same as static final fields).
Each constant is implicitly public static final — you get singleton instances for free.
enum Season {
SPRING, SUMMER, FALL, WINTER
}
Season now = Season.SUMMER;
🆓
Built-in Freebies
auto-generated
Every enum gets these for free — no extra code needed:
Season.SUMMER.name() // → "SUMMER"
Season.SUMMER.ordinal() // → 1 (0-indexed)
Season.valueOf("FALL") // → Season.FALL
Season.values() // → all 4, in order
Avoid leaning on ordinal() — adding a constant later silently shifts all positions. Use name() or a custom field instead.
③ Enums in action — real use cases
🔀
switch on Enums
best use case
Enums + switch are a perfect pair. The compiler warns you if you forget a case — that's exhaustiveness checking for free with modern Java.
switch (day) {
case MONDAY, TUESDAY ->
System.out.println("Grind time");
case SATURDAY, SUNDAY ->
System.out.println("Weekend!");
default -> ...
}
⚖️
Comparison & Equality
gotcha-proof
Enum constants are singletons, so == is safe and idiomatic — no .equals() required (unlike Strings!).
Enums implement Comparable automatically, ordered by declaration. You can sort a list of enum values with zero extra code.
if (season == Season.WINTER) {
// ✅ Safe! No .equals() needed
}
🗂️
EnumSet & EnumMap
performance win
EnumSet uses a single long bitmask under the hood — blazing fast, no boxing, purpose-built for enum flags.
EnumSet<Day> weekend =
EnumSet.of(Day.SAT, Day.SUN);
EnumMap<Day, String> plans =
new EnumMap<>(Day.class);
🛠️
Common Real-World Enums
use cases
State machines: OrderStatus.PENDING → SHIPPED → DELIVERED
Config flags: LogLevel.DEBUG / INFO / WARN / ERROR
HTTP verbs: HttpMethod.GET / POST / PUT / DELETE
UI modes: Theme.LIGHT / DARK / SYSTEM
The golden rule: if it's a closed, named set that rarely changes — reach for an enum.
⚡ Level Up — Enums can hold data too
Here's the twist: enums are secretly full Java classes. Each constant can carry constant-specific values, accessed via getters, and initialized through a private constructor. A planet enum could store mass and radius. A currency enum could store its symbol and decimal precision. That's next level — but the door is open.
fields + CTOR hint
enum Planet {
EARTH(5.97e24, 6.37e6),
MARS (6.39e23, 3.39e6);
final double mass, radius;
// private ctor + getters ...
}
why it matters
Instead of a separate lookup table, the data lives with the constant itself. Zero risk of the mapping going stale. Your enum IS the single source of truth.
