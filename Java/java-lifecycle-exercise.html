  1. Source Code — developer writes .java files
  2. Compilation (javac) — developer runs javac → produces .class bytecode
  3. JVM Loads Bytecode — JVM/HotSpot reads and verifies .class files
  4. Bytecode Interpretation — JVM interprets & executes bytecode line-by-line
  5. JIT Compilation — JVM JIT-compiles hot paths to native machine code
Java Execution Stack: .java → javac (AOT compile) → .class bytecode → JVM (HotSpot, written in C++) → interpreter → JIT → native machine code  |  Compare: Python uses PVM + C libraries; Java uses JVM + JIT for raw machine code performance.
