Anatomy of a Declaration

type — what kind of data this variable holds (int, double, String, …)  ·  identifier — the name you choose; use it to read or update the value  ·  = — the assignment operator; means "store this value here", not mathematical equality  ·  ; — every Java statement ends with a semicolon
Declaration Only
type + name · no value yet
// type name
int score;
double price;
boolean isActive;
// exists in memory, holds no value yet
Declaration + Initialization
declare and assign a starting value in one line
// type name = value
int score = 100;
double price = 9.99;
boolean isActive = true;
// the most common form you will write

Assignment & Reassignment

First Assignment
giving a declared variable its first value
int lives; // declared
 
lives = 3; // assigned
// no type keyword on assignment —
// just: name = value;
Reassignment
replacing the stored value with a new one
int lives = 3; // start: 3
lives = 2; // now: 2
lives = lives - 1; // now: 1
// type is NOT repeated on reassign
final — No Reassignment
assigned once · cannot be changed after init
final int MAX_LIVES = 3;
 
MAX_LIVES = 5; // ✗ compile error
// convention: name constants ALL_CAPS
// style

Naming Conventions

Rule ✓  Do this ✗  Avoid
camelCase for variables & methods firstName, totalScore, isActive FirstName, totalscore, IsActive
Start with a lowercase letter count, userName, maxSpeed Count, UserName, MaxSpeed
Use descriptive, readable names studentAge, itemCount, hasError a, x1, tmp2, thing
Constants: ALL_CAPS + final MAX_SIZE, DEFAULT_TIMEOUT, PI maxSize, defaultTimeout, pi
No leading underscores name, value, count _name, _value, __count
On leading underscores — Java does allow names like _count or __value, but avoid them. In other languages — C++, C#, Python — a leading underscore signals private members, name-mangled identifiers, or interpreter internals. That convention carries meaning there. In Java it carries no such meaning, only confusion — especially when you're reading or writing code alongside other languages. Stick to plain camelCase and save yourself the ambiguity.
Quick notes:   A variable must be declared before it can be used.  ·  Local variables must be initialized before they can be read — the compiler enforces this.  ·  The type keyword appears only once, at declaration. Reassignment uses just the name.  ·  Use final for constants and name them in ALL_CAPS.  ·  Java is strongly typed — a variable's type is fixed at declaration and cannot change.
